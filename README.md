## 📋 Описание проекта

SPA приложение для подачи заявки на займ, состоящее из трех последовательных форм с валидацией данных и финальным модальным окном подтверждения.

**Особенности:**
- ✅ Последовательное заполнение данных
- ✅ Валидация всех полей с подсказками
- ✅ Сохранение данных в localStorage (после перезагрузки данные не теряются)
- ✅ Навигация между формами с сохранением состояния
- ✅ Интеграция с тестовым API (dummyjson.com)
- ✅ Кэширование API запросов для оптимизации
- ✅ Единый стиль кода (Prettier)
- ✅ Линтинг и проверка качества кода (ESLint)

## 📖 Подробное обоснование выбора библиотек

### 1. React 19.2.5

**Почему выбран:** Современный, производительный, огромная экосистема

| Критерий | Обоснование |
|----------|-------------|
| **Компонентный подход** | Позволяет разбить формы на переиспользуемые компоненты (Input, Select, Button) |
| **Новый компилятор React 19** | Автоматически оптимизирует рендеры, уменьшая количество лишних перерисовок форм при вводе данных |
| **Hooks API** | Упрощает управление состоянием форм и валидацией через useState, useEffect |
| **Экосистема** | Огромное количество готовых решений и библиотек для любых задач |

### 2. TypeScript 6.0.2

**Почему выбран:** Безопасность типов, автодополнение, самодокументируемый код

| Критерий | Обоснование |
|----------|-------------|
| **Типизация пропсов** | Исключает ошибки передачи неправильных типов данных в компоненты |
| **Интерфейсы для API** | Четкое описание структуры ответов от сервера |
| **Автодополнение в IDE** | Ускоряет разработку и уменьшает количество опечаток |
| **Валидация форм** | Типизация ошибок валидации через Partial<FormData> |

### 3. React Router DOM 7.14.2

**Почему выбран:** Стандарт для навигации в React, поддержка кнопки "Назад"

| Критерий | Обоснование |
|----------|-------------|
| **Навигация без перезагрузки** | Переключение между формами происходит мгновенно, без потери состояния |
| **Кнопка "Назад" в браузере** | Пользователь может вернуться к предыдущей форме стандартными средствами |
| **URL маршруты** | `/` - форма 1, `/address` - форма 2, `/loan` - форма 3 |
| **Outlet компонент** | Позволяет создавать общий лейаут с хедером и футером |

### 4. Zustand 5.0.12

**Почему выбран:** Простой API, встроенный persist, минимальный бойлерплейт

| Критерий | Обоснование |
|----------|-------------|
| **Persist middleware** | Автоматически сохраняет данные форм в localStorage без дополнительного кода |
| **Минимальный код** | Для создания store нужно всего 10-15 строк кода |
| **TypeScript поддержка** | Отличная типизация из коробки |
| **Нет провайдера** | Не нужно оборачивать приложение в Provider |

### 5. TanStack Query 5.99.2

**Почему выбран:** Кэширование API, автоматические состояния загрузки, умное обновление

| Критерий | Обоснование |
|----------|-------------|
| **Кэширование запросов** | Список мест работы загружается 1 раз и кэшируется на 5 минут |
| **Автоматические состояния** | `isLoading`, `isError`, `data` - не нужно писать boilerplate код |
| **Retry при ошибке** | Автоматически повторяет запрос при ошибке сети |
| **DevTools** | Отличные инструменты для отладки запросов |

### 6. Axios 1.15.2

**Почему выбран:** Интерсепторы, единая обработка ошибок, удобная настройка

| Критерий | Обоснование |
|----------|-------------|
| **Интерсепторы** | Можно добавить токен авторизации ко всем запросам в одном месте |
| **Централизованная обработка ошибок** | Все ошибки API обрабатываются в одном месте |
| **baseURL** | Единый базовый URL для всех запросов через переменные окружения |
| **Timeout** | Автоматическая отмена запросов при превышении таймаута |

### 7. Styled Components 6.4.1

**Почему выбран:** CSS-in-JS, динамические стили, изоляция компонентов

| Критерий | Обоснование |
|----------|-------------|
| **Динамические стили** | Стили могут зависеть от props (например, `$isError` для подсветки ошибок) |
| **Изоляция стилей** | Стили компонента не влияют на другие компоненты |
| **Типизация** | TypeScript подсказывает доступные props для стилизованных компонентов |
| **Автоматическая префиксация** | Вендорные префиксы добавляются автоматически |

### 8. Vite 8.0.9

**Почему выбран:** Скорость, HMR, современные настройки

| Критерий | Обоснование |
|----------|-------------|
| **Быстрый запуск** | Сервер разработки запускается за < 500 мс |
| **HMR** | Изменения в коде видны мгновенно без перезагрузки страницы |
| **Алиасы (пути)** | `@/components` вместо `../../components` - чистый импорт |
| **Переменные окружения** | `import.meta.env` вместо `process.env` |

### 9. Prettier 3.8.3

**Почему выбран:** Автоматическое форматирование, единый стиль кода в команде

| Критерий | Обоснование |
|----------|-------------|
| **Единый стиль** | Все разработчики используют одинаковое форматирование (кавычки, отступы, запятые) |
| **Автоматизация** | Не нужно тратить время на ручное форматирование |
| **Интеграция с IDE** | Можно настроить автоформатирование при сохранении файла |
| **Pre-commit хуки** | Можно автоматически форматировать код перед коммитом (husky + lint-staged) |
| **Поддержка многих языков** | Форматирует JS, TS, JSON, CSS, SCSS, Markdown, HTML |
